# 标签页管理

本上下文定义一个 Chrome 标签页管理扩展中的核心领域语言。这里记录产品概念，不记录实现细节。

## 术语

**标签实例**:
浏览器普通窗口中当前真实存在的单个标签页。同一个规范化 URL 可以对应多个标签实例。
_避免使用_: 标签记录、页面

**内存指标**:
按需展示的浏览器进程级内存信息，用来辅助判断标签页资源开销。它不是单个标签页的精确独占内存。
_避免使用_: 标签页内存占用、精确内存

**标签记录**:
扩展按 URL 级别管理标签页的记录。一个标签记录可以对应一个或多个打开中的标签实例，也可以处于归档状态；同一个规范化 URL 不能同时既打开又归档。
_避免使用_: 标签实例、书签

**规范化 URL**:
用于识别标签记录、重复标签页、归档状态和侧边栏置顶的 URL 形式。第一版保留 origin、path、query，忽略 fragment。
_避免使用_: 原始 URL、完整 URL

**特殊 URL**:
浏览器内部页面、扩展页面、本地文件等非普通网页 URL。第一版可展示、可跳转，但不参与归档和休眠自动归档；重复检测可参与，自动重复处理应降级为提示。
_避免使用_: 普通网页、可归档页面

**重复标签页**:
在当前标签集合中，规范化 URL 与另一个打开标签页相同的标签页。重复检测跨所有普通窗口生效。
_避免使用_: 相同 URL 标签页、重复打开页面

**重复组**:
共享同一个规范化 URL 的打开中标签实例集合。重复组中的标签实例在列表中仍分行展示，但共享重复标识和数量。
_避免使用_: 合并标签、重复记录

**重复处理模式**:
用户选择的新页面重复处理方式。支持导航后提示，以及自动切换到最近激活的匹配标签实例并移除新打开的重复标签页；默认使用提示模式。
_避免使用_: 重复拦截、URL 拦截

**重复提示**:
在侧边栏内呈现的新页面重复提醒，提供跳转到已有标签页、保留当前标签页、查看重复实例等选择。第一版不使用网页内容注入或系统通知呈现重复提示；如果侧边栏未打开，提示模式只保留会话内待处理提醒，不主动打开侧边栏，且相关标签实例消失后清理提醒。
_避免使用_: 网页弹窗、系统通知

**归档标签页**:
用户主动把标签页移出当前浏览器窗口后形成的 URL 级非打开状态。它以规范化 URL 作为身份键，同时保留原始 URL 用于重新打开；它不是页面运行状态保存，也不是浏览器会话恢复；同一个规范化 URL 有打开实例时不能同时保留归档状态。
_避免使用_: 已保存标签、已关闭标签、会话恢复

**删除归档记录**:
移除扩展内某个归档标签记录的动作。它不删除浏览器历史、书签或任何网页数据。
_避免使用_: 删除历史、删除书签

**侧边栏置顶**:
用户对 URL 级标签记录设置的优先级标记，使该记录在侧边栏中跨打开和归档状态保持突出显示。它不改变 Chrome 原生标签页置顶状态。
_避免使用_: 原生置顶、浏览器置顶

**置顶区**:
侧边栏中展示已设置侧边栏置顶的标签实例和归档标签记录的独立区域。有打开实例时展示实例；没有打开实例时展示归档记录；使用用户手动排序，新置顶项默认放在最前；置顶区内的项不在普通分组列表中重复出现。
_避免使用_: 收藏夹、书签栏

**批量操作**:
对扩展中已选中的标签实例或规范化 URL 集合执行的操作。第一版批量操作包括归档、关闭、添加侧边栏置顶、移除侧边栏置顶、删除归档记录；删除归档记录只作用于已归档项。
_避免使用_: 批处理、批量命令

**跳转**:
把选中的打开中标签实例带到用户面前的动作，包括选中该标签页并聚焦其所在浏览器窗口。跳转到归档标签记录时，在当前普通窗口打开；如果没有普通窗口，则新建普通窗口，并移除其归档状态。
_避免使用_: 打开、导航

**关闭**:
移除一个打开中的标签实例，不创建或保留归档标签记录。
_避免使用_: 归档、忽略

**休眠标签页**:
长时间未被用户激活、且当前没有明确用户任务的普通标签页。第一版使用全局阈值，默认是距离上次激活超过七天。
_避免使用_: 僵尸标签页、无用标签页

**休眠自动归档**:
用户主动开启后，自动把休眠标签实例移出当前浏览器窗口的行为。保活、正在播放音频、Chrome 原生置顶的标签实例豁免；该行为默认关闭；只有当同一规范化 URL 没有剩余打开实例时，才形成归档状态。
_避免使用_: 自动关闭、内存清理

**最近自动归档记录**:
侧边栏中用于展示近期由休眠自动归档产生的归档动作的记录。它用于解释自动行为，不是完整操作日志；第一版只保留当前会话内最近 20 条。
_避免使用_: 审计日志、历史记录

**保活**:
用户给某个具体标签实例设置的休眠自动归档豁免。保活只作用于该标签实例，不作用于同一规范化 URL 的所有标签页，并在该标签实例关闭时结束；它不阻止用户手动归档或关闭。
_避免使用_: 白名单、置顶

**完整域名分组**:
基于标签页 URL 完整 host 的默认分组。不同子域名默认属于不同分组，除非用户通过自定义分组规则覆盖。
_避免使用_: 站点分组、根域名分组

**完整 host 匹配**:
自定义分组规则中的精确 hostname 匹配。它大小写不敏感，不包含端口。
_避免使用_: 域名匹配、host:port 匹配

**当前标签集合**:
当前所有普通浏览器窗口中打开的标签页集合。隐身窗口不属于本产品上下文。
_避免使用_: 所有标签页、会话

**配置范围**:
扩展观察标签页并保存用户管理状态的单个 Chrome profile。第一版不做跨设备同步，也不采集完整浏览历史。
_避免使用_: 账号同步、云同步

**窗口维度**:
作为标签页元信息和过滤条件使用的浏览器窗口信息。它不是标签清单的默认一级分组。
_避免使用_: 窗口分组、工作区

**浏览器标签组**:
Chrome 原生 Tab Group 信息，可作为标签页元信息展示并用于过滤。它独立于自定义分组，不决定标签清单的默认分组。
_避免使用_: 自定义分组、文件夹

**标签清单**:
侧边栏中的统一列表，包含打开中的标签实例和归档标签记录。归档标签默认包含在列表中，可通过过滤隐藏；归档项和打开项共享同一套分组规则，在同组中打开项排在归档项前；打开项按稳定窗口和 tab index 排序，避免点击后列表跳变；归档项按最近归档时间排序。
_避免使用_: 当前标签列表、标签会话

**搜索**:
对标签标题、完整域名、规范化 URL、自定义分组名称进行文本匹配。
_避免使用_: 过滤、查询

**过滤**:
对标签清单状态或元信息施加的结构化约束，例如打开、归档、休眠、重复、保活、正在播放音频、Chrome 原生置顶、窗口、浏览器标签组。
_避免使用_: 搜索、文件夹

**自定义分组**:
用户定义的规则型分组，可跨完整域名分组收集标签页。自定义分组名称在当前配置范围内唯一；每个自定义分组可包含多条匹配规则，组内任意规则命中即可归入该组；自定义分组按用户定义的优先级顺序执行，首个命中的自定义分组胜出，未命中时回落到完整域名分组。
_避免使用_: 文件夹、分类

**URL 模式**:
用于分组的正则表达式，匹配对象是规范化 URL；第一版默认大小写不敏感。
_避免使用_: URL 包含、通配符
