# 产品设计

## 产品目标

这个扩展解决两个核心问题：

1. 标签页太多，用户难以理解当前浏览器里打开了什么。
2. 久未使用的标签页继续占用浏览器资源，增加性能负担。

第一版产品聚焦单个 Chrome profile 内的普通窗口，不处理隐身窗口，不做跨设备同步，不记录完整浏览历史。

## 核心体验

用户通过 Chrome 侧边栏查看标签清单。标签清单默认包含当前打开的标签实例和已归档的标签记录，用户可以通过过滤隐藏归档项。

侧边栏顶部有置顶区。置顶区展示用户设置了侧边栏置顶的标签记录对应内容：如果当前有打开实例，展示实例；如果没有打开实例但存在归档状态，展示归档记录。置顶区使用用户手动排序，新置顶项默认在最前，置顶项不在普通分组列表中重复展示。

普通列表按照自定义分组或完整域名分组展示。自定义分组优先于完整域名分组，自定义分组按用户配置顺序执行，首个命中胜出。每个自定义分组可以包含多条匹配规则，组内任意规则命中即可归入该组。未命中自定义分组的标签回落到完整 host 分组。

每个自定义分组和每条自定义分组规则都有启用状态。禁用后保留配置，但不参与匹配。

自定义分组名称在当前 Chrome profile 内必须唯一。

自定义分组使用用户手动排序。新建自定义分组默认放在优先级列表顶部。

组内规则没有优先级，任意启用规则命中即可。

每个分组标题显示数量摘要，包括总项数、打开项数、归档项数和重复项数。休眠数量优先通过过滤器或状态标识呈现，不默认塞进分组标题。

MVP 普通分组按组内总项数倒序排列；数量相同时按完整 host 字母序排列。这样最拥挤的域名优先出现在用户视野中。

普通分组默认展开。用户手动折叠或展开分组后，侧边栏记住该状态；新出现的分组默认展开。

搜索或过滤激活时，侧边栏临时展开包含命中项的分组，并隐藏没有命中项的分组。退出搜索或过滤后，恢复用户原本的折叠状态。

标签实例行以标题作为主文本，使用完整域名和路径的紧凑形式作为辅助文本。完整 URL 通过详情展开或悬浮提示查看，避免长 query 直接污染主列表。

标签实例行可以展示内存指标，但该指标是可选显示，不默认持续实时刷新。内存指标用于辅助判断资源开销，不承诺是单个标签页的精确独占内存；如果多个标签页共享同一浏览器进程，可能显示相同或相关的进程级数据。

## 标签页身份模型

产品同时保留两个层级：

- 标签实例：浏览器中真实存在的单个 tab，用于展示、跳转、关闭、保活、窗口信息等实例级操作。
- 标签记录：按规范化 URL 管理的 URL 级状态，用于归档、侧边栏置顶、重复组等 URL 级能力。

规范化 URL 第一版保留 origin、path、query，忽略 fragment。这个规则用于重复检测、归档状态、侧边栏置顶和自定义分组 URL 模式匹配。

同一个规范化 URL 不能同时处于打开和归档状态。如果存在打开实例，则该标签记录是打开状态；只有没有任何打开实例时，才可以处于归档状态。

## 重复标签页

重复标签页指当前标签集合中规范化 URL 相同的打开标签实例，检测范围跨所有普通窗口。

重复组在列表中不合并为单行。每个标签实例仍然独立展示，但共享重复标识和数量。用户可以通过重复标识或过滤器聚焦查看同一重复组。

当用户打开新页面且命中重复 URL 时，重复处理模式有两种：

- 提示模式：默认模式。导航后提示用户已有相同页面，可选择跳转到已有标签页或保留当前标签页。
- 自动模式：自动切换到最近激活的匹配标签实例，并移除新打开的重复标签页。

第一版重复提示只在侧边栏内呈现，不做网页内容注入，也不使用系统通知。提示中提供跳转到已有标签页、保留当前标签页、查看所有重复实例等动作。

如果侧边栏当前未打开，提示模式不主动打开侧边栏，只记录一个待处理的重复提示，等用户打开侧边栏时展示。自动模式不依赖侧边栏，仍按用户配置直接跳转并移除新重复标签页。

待处理重复提示只保留在当前浏览器会话内，不跨浏览器重启持久化。如果相关的新重复标签页已经被关闭或处理，对应提示自动清理。

如果已有多个重复实例，默认目标是最近激活的实例；提示模式中允许用户展开选择其他实例。用户选择“保留当前重复标签页”只对当次生效，第一版不提供长期重复豁免规则。

## 归档

归档是状态切换，不是收藏库，也不是完整会话恢复。

用户归档一个标签实例时，该实例会从浏览器窗口移除。如果同一规范化 URL 仍有其他打开实例，则标签记录仍保持打开状态；如果没有剩余打开实例，则进入归档状态。

归档记录以规范化 URL 作为身份键，同时保存原始 URL 用于重新打开。这样 hash 不参与重复身份判断，但恢复时仍尽量打开用户当时的原始地址。

如果同一规范化 URL 的多个实例只有原始 URL 差异，例如 hash 不同，则归档记录保存最后被归档的那个标签实例的原始 URL。

归档记录保存最后归档实例的标题和 favicon URL，用于归档状态下展示。打开状态下优先展示当前标签实例的实时标题和 favicon。标题和 favicon 不参与标签身份判断。

归档记录可以显示最后来源窗口作为历史上下文，但恢复时默认在当前普通窗口打开，不强制回到来源窗口。

用户跳转到归档标签记录时，会在当前普通窗口打开该 URL；如果没有普通窗口，则创建普通窗口。打开后，该标签记录离开归档状态。

用户可以删除归档记录。删除归档记录只移除扩展内的归档状态，不影响浏览器历史、书签或网页数据。

第一版不自动过期清理归档记录。归档记录只会因为用户打开该归档项或显式删除归档记录而消失。

关闭和归档是两个不同动作。关闭只移除打开中的标签实例，不创建或保留归档记录。

第一版不提供关闭或归档后的撤销能力。

## 休眠自动归档

休眠标签页按最后激活时间判断，第一版使用全局阈值，默认是七天。归档项不参与休眠判断，因为它已经不占用当前浏览器资源。

扩展安装后首次观察到的已有标签页，以首次观察时间作为最后激活时间基准，不追溯安装前的使用历史，也不在安装后立即判定为休眠。

休眠自动归档是可选能力，默认关闭。用户主动开启后，扩展可以自动把休眠标签实例移出浏览器窗口。

以下标签实例不参与休眠自动归档：

- 用户设置了保活的实例
- 正在播放音频的实例
- Chrome 原生置顶的实例

保活只影响休眠自动归档，不阻止用户手动归档或关闭该标签实例。

第一版不承诺检测未提交表单或编辑器未保存内容。风险通过默认关闭、明确开启、保活豁免和可靠浏览器信号控制。

休眠自动归档执行前不做逐项预告。自动归档发生后，侧边栏展示最近自动归档记录，帮助用户理解哪些标签页被自动归档。

最近自动归档记录只保留当前浏览器会话内最近 20 条，不跨浏览器重启持久化。

## 搜索与过滤

搜索用于文本匹配，匹配范围包括标签标题、完整域名、规范化 URL、自定义分组名称。

过滤用于结构化条件，包括打开、归档、休眠、重复、保活、正在播放音频、Chrome 原生置顶、窗口、浏览器标签组。

窗口维度和 Chrome 原生 Tab Group 只作为元信息和过滤条件，不作为默认一级分组，也不参与自定义分组规则。

## 特殊 URL

`chrome://`、扩展页面、本地文件等特殊 URL 第一版纳入展示和跳转，但不参与归档和休眠自动归档。特殊 URL 可以参与重复检测，但自动重复处理应降级为提示模式。

## 批量操作

第一版支持批量归档、批量关闭、批量添加侧边栏置顶、批量移除侧边栏置顶、批量删除归档记录。批量删除归档记录只作用于已归档项；如果选择集合中包含打开中的标签实例，应跳过这些打开项并向用户说明跳过数量。

分组标题支持选择该组当前可见项。搜索或过滤激活时，分组级选择只选择当前结果中的可见项，不选择被过滤隐藏的项。

批量归档时，特殊 URL 跳过；正在播放音频和 Chrome 原生置顶的标签实例需要二次确认；保活实例不阻止手动批量归档。

批量关闭时，特殊 URL 可以关闭但必须计入确认摘要；正在播放音频、Chrome 原生置顶、保活、最近五分钟激活过的标签实例需要二次确认。

单个关闭或归档默认不二次确认。单个关闭正在播放音频或 Chrome 原生置顶的标签实例时需要确认；单个归档正在播放音频或 Chrome 原生置顶的标签实例时也需要确认。特殊 URL 不允许归档，但允许单个关闭。

## 不做的事

第一版明确不做：

- 不管理隐身窗口
- 不做跨设备同步
- 不记录完整浏览历史
- 不保存页面内容、滚动位置、表单状态或登录态
- 不把归档当成浏览器会话恢复
- 不自动过期清理归档记录
- 不把 Chrome 原生 Tab Group 和自定义分组同步
- 不提供长期重复 URL 豁免规则
- 不支持按页面标题配置自定义分组规则
