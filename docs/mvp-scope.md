# MVP 范围

本文档定义最小可用版本范围。MVP 的目标是先验证标签清单、重复识别、基础归档和基础整理工作流，不一次性实现完整第一版规划。

状态：已冻结，当前实现已完成。后续新增能力默认进入 MVP 之后，除非用户明确解冻并修改本文档。

## MVP 目标

MVP 要让用户能够：

1. 在侧边栏看清当前普通窗口中打开的标签页。
2. 识别重复标签页。
3. 跳转、关闭、归档标签页。
4. 查看和恢复归档项。
5. 使用搜索和基础过滤快速缩小列表范围。

## 必须包含

### 数据存储

- 使用 `chrome.storage.local` 保存归档记录和分组折叠状态。
- 不使用 `chrome.storage.sync`。

### 侧边栏标签清单

- 展示单个 Chrome profile 内所有普通窗口的打开中标签实例。
- 排除隐身窗口。
- 标签行标题优先展示，辅助展示完整域名和路径紧凑形式。
- 分组标题展示 favicon；同 host 使用组级 icon，缺失时显示通用占位图标。
- 展示最小窗口标识，用于说明标签实例所在窗口。
- 当前选中的标签页高亮展示，并在打开或切换到侧边栏时锚定到可视位置。
- 完整 URL 可通过底部固定 URL inspector 查看，不依赖悬浮提示。
- 按完整 host 默认分组。
- 普通分组按组内总项数倒序排列；数量相同时按完整 host 字母序排列。
- 分组标题显示总项数、打开项数、归档项数和重复项数。
- 普通分组默认展开，支持手动折叠状态保留。

### 重复识别

- 使用规范化 URL 判断重复，保留 origin、path、query，忽略 fragment。
- 重复检测跨所有普通窗口。
- 重复标签实例不合并，仍分行展示。
- 重复组显示共享重复标识和数量。
- 支持重复过滤。

### 基础操作

- 跳转到打开中的标签实例，并聚焦所在窗口。
- 关闭单个标签实例。
- 归档单个普通网页标签实例。
- 特殊 URL 可展示和跳转，但不允许归档。
- 关闭和归档保持独立，关闭不创建归档记录。
- 关闭和归档都不提供撤销。
- 单个关闭不做二次确认。
- 单个归档不做二次确认。
- 行内操作按钮默认常显，不依赖 hover 才出现。
- 行点击作为跳转或恢复的主操作；行内按钮保留归档、关闭、删除等次要操作。

### 归档

- 归档记录以规范化 URL 作为身份键，同时保存原始 URL 用于恢复。
- 归档记录保存最后归档实例的标题和 favicon URL，favicon 用于后续组级 icon fallback。
- 归档记录显示最后来源窗口作为历史上下文。
- 同一规范化 URL 有打开实例时，不保留归档状态。
- 打开归档项后，在当前普通窗口恢复，并移除归档状态。
- 删除单个归档记录。
- 归档项默认包含在标签清单中，并可通过过滤隐藏。

### 搜索与过滤

- 搜索匹配标题、完整域名、规范化 URL。
- 搜索同时匹配打开中的标签实例和归档标签记录。
- 支持单一状态过滤：全部、打开、归档、重复。
- 搜索可以和当前状态过滤同时生效。
- 搜索词不持久化。
- 状态过滤可在当前侧边栏会话内保留，但不跨浏览器重启。

### 空状态

- 没有普通窗口标签页时，展示空状态。
- 搜索或过滤无结果时，展示空状态。
- 没有归档项时，归档过滤展示空状态。
- 搜索或过滤激活时，临时展开命中分组并隐藏无命中分组。
- 退出搜索或过滤后恢复原分组折叠状态。

### 加载和错误状态

- 首次加载标签清单时展示 loading 状态。
- Chrome API 读取失败时展示 error 状态。
- 跳转、关闭、归档、恢复、删除归档记录失败时，在侧边栏内展示非阻塞错误提示。

### 基础无障碍

- 所有操作按钮有可读标签。
- 列表项和主要操作可通过键盘聚焦。
- 重复、归档、错误、空状态不只依赖颜色表达。

## MVP 不包含

- 自定义分组配置。
- 侧边栏置顶和置顶区。
- 批量操作。
- 新页面重复提示和自动重复处理。
- 休眠标签页识别。
- 休眠自动归档。
- 保活。
- Chrome 原生 Tab Group 过滤。
- 窗口过滤。
- 最近自动归档记录。
- 内存指标。
- 独立设置页。
- 关闭或归档后的撤销能力。
- 单个关闭的高风险确认规则。
- 单个归档的高风险确认规则。
- 自定义键盘快捷键。
- 跨设备同步。
- 完整浏览历史记录。
- 隐身窗口管理。

## MVP 验收场景

### 查看当前标签

给定用户打开多个普通 Chrome 窗口，每个窗口有多个标签页；打开侧边栏后，应能看到所有普通窗口中的标签实例，且不包含隐身窗口标签页。

### 识别重复标签

给定用户打开两个 URL 仅 hash 不同的标签页；侧边栏应将它们识别为重复，并显示重复标识和数量。

给定用户打开两个 query 不同的标签页；侧边栏不应将它们识别为重复。

### 跳转标签

给定用户点击侧边栏中的打开中标签实例；扩展应选中该标签页，并聚焦其所在 Chrome 窗口。

### 窗口标识

给定用户打开多个普通 Chrome 窗口；侧边栏应能让用户识别每个标签实例所在窗口，但不需要支持按窗口过滤。

### 归档标签

给定用户归档一个普通网页标签实例；该标签实例应从浏览器窗口移除，并在侧边栏中出现归档项。

给定同一规范化 URL 还有其他打开实例；归档其中一个实例后，不应创建归档状态。

### 恢复归档

给定用户点击归档项；扩展应在当前普通窗口打开该归档项保存的原始 URL，并移除归档状态。

归档项可以显示最后来源窗口，但恢复时不强制回到来源窗口。

### 删除归档

给定用户删除归档记录；扩展只删除自己的归档状态，不影响浏览器历史或书签。

### 搜索过滤

给定用户输入搜索词；侧边栏只展示匹配标题、完整域名或规范化 URL 的项，并临时展开命中分组。

搜索结果应同时覆盖打开项和归档项。

给定用户开启归档过滤；侧边栏应能隐藏或只显示归档项，具体过滤 UI 在实现阶段确定。

MVP 不需要支持多个状态的组合过滤；搜索词可以叠加在当前状态过滤上。

搜索词不跨会话保留；状态过滤最多保留在当前侧边栏会话内。

### 空状态

没有普通窗口标签页、搜索或过滤无结果、没有归档项时，侧边栏应展示可理解的空状态，而不是空白界面。

### 加载和错误状态

首次读取标签清单时应显示 loading。Chrome API 读取失败或用户操作失败时，应在侧边栏内显示可理解的错误提示，且不导致整个侧边栏不可用。

### 基础无障碍

操作按钮、列表项和状态提示应可被辅助技术理解；重复、归档、错误等状态不能只靠颜色区分。
