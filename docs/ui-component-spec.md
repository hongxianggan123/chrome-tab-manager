# UI 组件规格

本文档把 MVP UI 原型拆成可实现的组件规格。它约束界面结构、状态、文案和交互，不包含前端代码。

实现阶段使用 shadcn/ui 作为基础组件系统，具体组件映射见 [shadcn/ui 组件计划](./shadcn-component-plan.md)。

## 组件地图

MVP 侧边栏由以下组件组成：

```text
SidePanel
├─ PanelHeader
├─ SearchBox
├─ StatusFilter
├─ DuplicateCleanupAction
├─ FeedbackRegion
├─ GroupList
│  ├─ GroupSection
│  │  ├─ GroupHeader
│  │  └─ InventoryRow
│  └─ EmptyState
```

## SidePanel

职责：

- 提供整个侧边栏的布局容器。
- 控制 loading、error、empty、content 四类主状态。
- 保持顶部搜索和过滤区域固定在上方，列表区域滚动。
- 管理当前 active tab 的可视锚定。

尺寸建议：

- 默认宽度按 360px 设计。
- 最小宽度 320px。
- 顶部控制区不超过 176px，避免挤压列表。

结构：

```text
┌────────────────────────────┐
│ PanelHeader                │
│ SearchBox                  │
│ StatusFilter               │
│ DuplicateCleanupAction     │
│ DuplicatePromptBanner      │
├────────────────────────────┤
│ FeedbackRegion             │
├────────────────────────────┤
│ GroupList                  │
└────────────────────────────┘
```

## PanelHeader

职责：

- 显示产品当前范围和总量。
- 不做营销文案，不解释功能。

内容：

- 主标题：`Tabs`
- 总数：当前过滤前的标签清单总项数。
- 副标题：`普通窗口 · 归档已包含`
- 设置入口：右侧齿轮图标按钮，label 为 `打开设置`。

示例：

```text
Tabs                                  42 [⚙]
普通窗口 · 归档已包含
```

状态：

- loading 时总数可显示 `--`。
- error 时总数隐藏。

行为：

- 点击设置入口进入 `SettingsPanel` 设置视图。
- 设置入口始终可见，避免未来配置入口散落到多个局部控件中。

## SettingsPanel

职责：

- 承载侧边栏内的可扩展设置。
- 当前只包含重复提示展示方式，后续按能力分区增加配置。

位置：

- 由 `PanelHeader` 右侧齿轮图标进入。
- 替换主列表区域，不使用叠层抽屉，不新建 options page。
- 设置视图顶部提供返回按钮，label 为 `返回标签清单`。

结构：

```text
< 返回  设置
重复提示
  展示方式
  ( ) 侧边栏
  ( ) 页面浮层
```

行为：

- 设置按分区组织，第一组为 `重复提示`。
- `重复提示展示方式` 使用单选控件，选项为 `侧边栏` 和 `页面浮层`。
- 选择 `页面浮层` 时立即请求页面浮层所需授权。
- 授权成功后保存设置；授权拒绝或失败时保持或回退到 `侧边栏`，并显示非阻塞错误。
- 如果页面浮层授权之后被用户关闭或撤销，设置自动回到 `侧边栏`，并显示 `页面浮层权限已关闭，已改用侧边栏提示。`
- 设置变更即时生效，不需要保存按钮。
- 点击返回回到标签清单，并保留原有搜索、过滤和选择状态。

视觉：

- 工具型、紧凑、分区清晰，不使用营销式说明。
- 使用安静的标题、细分隔线和标准表单控件，避免卡片嵌套。
- 不使用 `Sheet` 或浮层抽屉，避免窄侧边栏里出现层叠滚动。
- 页面浮层授权说明使用一行辅助文本，说明“只用于显示重复提示，不读取网页内容”。

## SearchBox

职责：

- 搜索标题、完整域名、规范化 URL。
- 搜索打开项和归档项。

占位文案：

```text
搜索标题、域名或 URL
```

行为：

- 输入后立即过滤。
- 搜索词不持久化。
- 清空搜索后恢复当前状态过滤下的结果。
- 搜索激活时，命中分组临时展开，无命中分组隐藏。

无障碍：

- 输入框需要可读 label：`搜索标签页`。
- 清空按钮需要可读 label：`清空搜索`。

## StatusFilter

职责：

- 提供单一状态过滤。
- 搜索词可以叠加当前过滤。

选项：

- `全部`
- `打开`
- `归档`
- `重复`

样式：

- 使用 segmented control。
- 每个选项展示当前搜索条件下的计数。

示例：

```text
[全部 42] [打开 36] [归档 4] [重复 2]
```

行为：

- 同一时间只能选中一个状态。
- 状态过滤最多保留在当前侧边栏会话内，不跨浏览器重启。

## DuplicateCleanupAction

职责：

- 快速选中当前可见结果中可清理的重复标签实例。
- 只产生批量选择，不直接关闭、归档或删除。

位置：

- 位于顶部控制区中，放在 StatusFilter 下方。
- 使用右对齐、内容宽度的轻量胶囊，不占满整行。

内容：

```text
                         [ 选中重复 3 ]
```

行为：

- 点击后替换当前选择集合，并清除待确认的批量动作。
- 搜索或过滤激活时，只根据当前可见结果计算目标。
- 每个可见重复组保留一个重复组保留实例，选中其余可见打开项。
- 没有可选目标时保留入口但 disabled，文案为 `无重复`，避免顶部布局跳动。
- `保留 N 个最新` 作为可访问标签和浏览器 title 提供，不在顶部常显。

视觉：

- 左侧使用重复语义图标，不使用竖向重复轨道。
- 使用重复状态的琥珀描边和 hover 底色，但默认保持透明背景，避免抢占搜索和过滤入口。

## DuplicatePromptBanner

职责：

- 展示最近一条待处理重复提示。
- 让用户在跳转到已有标签页、保留当前标签页、查看重复实例之间做选择。

位置：

- 位于顶部控制区下方、标签清单上方。
- 不放入底部反馈区或批量操作栏，避免和批量确认状态重叠。

内容：

```text
已打开重复页面
新页面标题 · example.com
已有目标：目标标题 · W1
[跳转] [保留] [查看重复]
```

行为：

- 同一时间只展示一条最近重复提示。
- 点击 `跳转` 后跳转到默认目标，并关闭触发提示的新重复标签页。
- 点击 `保留` 后关闭提示，保留新重复标签页。
- 点击 `查看重复` 后关闭提示，清空搜索，切换到重复过滤，并优先定位触发提示的新重复标签页；如果它已不存在，则定位默认目标。
- 三个动作执行后都关闭当前提示。
- 提示展示 30 秒后自动关闭；最后 5 秒在 `保留` 按钮上展示倒计时。
- 自动关闭不关闭新重复标签页，但同一个标签实例在当前会话内不再反复提示。

视觉：

- 使用紧凑 inline banner，不阻塞列表滚动。
- 使用重复状态的琥珀语义色作为辅助，不只依赖颜色表达。
- 按钮使用紧凑尺寸，保证窄侧边栏下文本不溢出。

## DuplicatePromptOverlay

职责：

- 在用户开启页面浮层并完成授权后，在普通网页中展示最近一条重复提示。
- 让用户不打开侧边栏也能处理新打开的重复标签页。

位置：

- 注入到触发重复提示的新重复标签页页面中。
- 默认放在页面右上角可见区域，避开浏览器自身 UI。
- 固定定位，距离视口右侧和顶部各 16px；窄屏下宽度不超过 `calc(100vw - 32px)`。
- 不在 `chrome://`、Chrome Web Store、扩展页面等不可注入页面展示。

内容：

```text
已打开重复页面
已有目标：目标标题 · W1
[跳转] [保留] [查看重复]
```

行为：

- 与 `DuplicatePromptBanner` 使用同一组动作语义。
- 第一版不支持拖动，也不记住浮层位置。
- 点击 `跳转` 后跳转到默认目标，并关闭当前新重复标签页。
- 点击 `保留` 后关闭浮层，保留当前新重复标签页。
- 点击 `查看重复` 后打开侧边栏，关闭浮层，清空搜索，切换到重复过滤，并优先定位当前新重复标签页；如果它已不存在，则定位默认目标。
- 三个动作执行后都关闭当前浮层。
- 浮层展示 30 秒后自动关闭；最后 5 秒在 `保留` 按钮上展示倒计时。
- 自动关闭不关闭新重复标签页，但同一个标签实例在当前会话内不再反复提示。
- 注入失败、授权缺失或页面不可注入时，降级为侧边栏待处理提示。
- service worker 挂起恢复后，已自动关闭或已处理的同一 tabId 不应再次出现浮层。

视觉：

- 使用 Shadow DOM 和扩展专用命名空间，避免受网页 CSS 影响。
- 宽度 300-340px，尺寸保持紧凑，不遮挡页面主要内容。
- 使用白色或接近系统面板色的背景、细边框和轻阴影，避免像网页广告或系统通知。
- 顶部使用一条 3px 琥珀色重复状态条作为唯一强视觉记号，不使用大面积彩色背景。
- 标题使用 `已打开重复页面`，正文展示目标标题和窗口标识；长标题单行截断。
- 按钮从左到右为 `跳转`、`保留`、`查看重复`；`跳转` 是主按钮，`保留` 和 `查看重复` 是次级按钮。
- 最后 5 秒将 `保留` 按钮文案改为 `保留 5`、`保留 4` 直到自动关闭。
- 入场动效只使用 120ms 以内的轻微 opacity/translate，尊重 `prefers-reduced-motion`。
- 提供清晰可聚焦按钮和可访问标签。
- 不读取网页正文，不扫描页面 DOM。

## FeedbackRegion

职责：

- 展示非阻塞反馈。
- 不覆盖列表，不阻止用户继续操作。

类型：

- 操作失败
- API 读取失败后的重试提示
- 特殊 URL 不可归档提示

文案规则：

- 说明发生了什么。
- 说明用户能做什么。
- 不使用泛化文案，如 `出错了`。

示例：

```text
无法归档 chrome://extensions。特殊 URL 只能跳转或关闭。
```

```text
无法关闭这个标签页。它可能已经被关闭，请刷新列表。
```

## GroupList

职责：

- 展示按完整 host 分组后的标签清单。
- 承载空状态。

滚动：

- 只有 GroupList 滚动，顶部搜索和过滤保持可见。
- 分组标题在 GroupList 滚动容器内吸顶，背景保持不透明，并通过下沿分隔与滚动中的行区分。

排序：

- 分组排序第一版按组内总项数倒序；数量相同按 host 字母序。
- 组内打开项排在归档项前。
- 打开项按稳定窗口 id 和 tab index 排序，不按最近激活时间排序，避免点击后列表跳变。
- 归档项按最近归档时间倒序。

说明：

- MVP 采用“负载优先”的分组排序。标签页最多的域名最需要用户处理。

## GroupHeader

职责：

- 展示分组名称、数量摘要、折叠状态。
- 展示组级 favicon；同 host 的行共享分组 icon。
- 当分组内包含当前 active tab 时，展示轻量当前提示。
- 支持展开/折叠。

内容：

```text
▾ docs.google.com
8 项 · 打开 6 · 归档 1 · 重复 2
```

行为：

- 点击分组标题展开/折叠。
- 用户折叠状态持久化到 `chrome.storage.local`。
- 搜索或过滤激活时，命中分组临时展开。
- 退出搜索或过滤后恢复用户折叠状态。

无障碍：

- 分组标题是 button。
- 需要表达 expanded/collapsed 状态。

## InventoryRow

职责：

- 展示一个打开中的标签实例，或一个归档标签记录。
- 提供该行可用操作。

### 打开项

结构：

```text
│标题                              │
│当前 W2 重复 x2           归档 关闭│
```

字段：

- 标题
- 当前 active tab 标识
- 窗口标识
- 重复徽标
- 操作按钮

可用操作：

- 点击行跳转
- hover 或 focus 到行左缘选择命中区时显示从左侧滑入的悬浮选择控件；选中后控件停在同一终点位置常显，正文缩进进入批量选择态。
- 归档，特殊 URL 不显示或禁用
- 关闭

### 归档项

结构：

```text
│API notes                         │
│已归档 · 来自 W2             删除 │
```

字段：

- 归档时保存的标题
- 最后来源窗口
- 归档状态

可用操作：

- 点击行打开并恢复归档项
- hover 或 focus 到行左缘选择命中区时显示从左侧滑入的悬浮选择控件；选中后控件停在同一终点位置常显，正文缩进进入批量选择态。
- 删除归档记录

## 悬浮选择槽

职责：

- 在不牺牲列表横向空间的前提下提供批量选择入口。
- 让行和分组标题使用一致的选择语义。

规格：

- 默认不占据布局列，正文保持正常左边距。
- hover 或 focus 到行/分组标题左缘选择命中区时，显示小尺寸圆形选择控件和整行高度的矩形渐变遮罩；选择控件从左侧滑入，离开后滑出。
- 选中后选择控件在相同位置常显，不显示遮罩，正文左边距过渡到批量选择宽度；选择控件在缩进空间内保持左右间距一致。
- 分组部分选中时使用半选图标。

状态表达：

- 当前、打开、归档、重复和特殊 URL 通过 badge 与行底色表达。
- 选择控件只表达批量选择状态，不承担业务状态表达。

无障碍：

- 选择控件是可聚焦 button，并提供可读 label。
- 选中、半选和未选状态通过 `aria-pressed` 表达。
- 重复、归档、特殊 URL 必须有文字徽标或 label；颜色只是辅助。

## 当前页高亮

职责：

- 让用户知道当前 Chrome active tab 在清单中的位置。
- 切换 active tab 后自动把当前行滚动到列表可视区域。

行为：

- 当前行设置 `aria-current="page"`。
- 当前行展示 `当前` 徽标。
- 当前行使用轻量背景和边框突出。
- 当前行所属分组标题展示 `当前` 提示。
- 自动滚动只发生在当前项变化时。

## 完整 URL 查看

当前侧边栏主列表暂不展示完整 URL 或紧凑 URL，也不使用底部 URL inspector。后续如果重新加入完整 URL 查看能力，需要单独设计入口，避免占用高密度列表空间。

## 操作按钮

MVP 操作按钮默认常显，不依赖 hover 才出现。按钮使用图标按钮，必须有可读 `aria-label`。tooltip 不作为按钮说明的主要机制。

按钮：

- 归档：`归档标签页`
- 关闭：`关闭标签页`
- 删除归档记录：`删除归档记录`

点击范围：

- 最小 28px × 28px。
- 行内视觉可以紧凑，但实际点击区域不能低于该尺寸。

按钮排列：

```text
打开项：归档 / 关闭
归档项：删除
特殊 URL：关闭
```

## EmptyState

### 没有普通窗口标签页

标题：

```text
没有可管理的普通窗口标签页
```

说明：

```text
打开一个普通 Chrome 窗口后，标签页会显示在这里。
```

### 搜索或过滤无结果

标题：

```text
没有匹配的标签页
```

说明：

```text
试试搜索域名、标题里的关键词，或清空当前过滤。
```

### 没有归档项

标题：

```text
还没有归档项
```

说明：

```text
归档一个普通网页后，它会出现在这里。
```

## LoadingState

首次加载文案：

```text
正在读取当前标签页...
```

展示方式：

- 顶部控制区可以先显示骨架。
- 列表区显示 4 到 6 条行骨架。
- 不使用大面积 spinner。

## ErrorState

主错误：

```text
无法读取标签页
Chrome 没有返回当前标签页列表。
[重试]
```

非阻塞错误：

- 出现在 FeedbackRegion 或 ToastRegion。
- 5 秒后自动消失。
- 用户也可以手动关闭。

| 场景 | 文案 |

| 搜索占位 | 搜索标题、域名或 URL |
| 全部过滤 | 全部 |
| 打开过滤 | 打开 |
| 归档过滤 | 归档 |
| 重复过滤 | 重复 |
| 重复徽标 | 重复 x{count} |
| 归档状态 | 已归档 |
| 来源窗口 | 来自 W{number} |
| 特殊 URL | 特殊 URL |
| 归档按钮 | 归档标签页 |
| 关闭按钮 | 关闭标签页 |
| 删除归档记录 | 删除归档记录 |

说明：

- 状态徽标使用中文短标签，保持界面语言一致。
- 操作按钮使用中文 `aria-label`，降低误操作。

## UI 决策状态

MVP UI 组件规格已冻结且当前实现已完成。后续新增交互或改变按钮可见性前需要先更新本文档。
