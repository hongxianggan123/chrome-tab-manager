# UI 组件规格

本文档把 MVP UI 原型拆成可实现的组件规格。它约束界面结构、状态、文案和交互，不包含前端代码。

实现阶段使用 shadcn/ui 作为基础组件系统，具体组件映射见 [shadcn/ui 组件计划](./shadcn-component-plan.md)。

## 组件地图

MVP 侧边栏由以下组件组成：

```text
SidePanel
├─ PanelHeader
├─ SearchBox
├─ StatusFilter
├─ FeedbackRegion
├─ GroupList
│  ├─ GroupSection
│  │  ├─ GroupHeader
│  │  └─ InventoryRow
│  └─ EmptyState
└─ UrlInspector
```

## SidePanel

职责：

- 提供整个侧边栏的布局容器。
- 控制 loading、error、empty、content 四类主状态。
- 保持顶部搜索和过滤区域固定在上方，列表区域滚动。
- 管理当前 active tab 的可视锚定。
- 管理底部固定 URL inspector 的预览和锁定状态。

尺寸建议：

- 默认宽度按 360px 设计。
- 最小宽度 320px。
- 顶部控制区不超过 132px，避免挤压列表。

结构：

```text
┌────────────────────────────┐
│ PanelHeader                │
│ SearchBox                  │
│ StatusFilter               │
├────────────────────────────┤
│ FeedbackRegion             │
├────────────────────────────┤
│ GroupList                  │
├────────────────────────────┤
│ UrlInspector                │
└────────────────────────────┘
```

## PanelHeader

职责：

- 显示产品当前范围和总量。
- 不做营销文案，不解释功能。

内容：

- 主标题：`Tabs`
- 总数：当前过滤前的标签清单总项数。
- 副标题：`普通窗口 · 归档已包含`

示例：

```text
Tabs                                      42
普通窗口 · 归档已包含
```

状态：

- loading 时总数可显示 `--`。
- error 时总数隐藏。

## SearchBox

职责：

- 搜索标题、完整域名、规范化 URL。
- 搜索打开项和归档项。

占位文案：

```text
搜索标题、域名或 URL
```

行为：

- 输入后立即过滤。
- 搜索词不持久化。
- 清空搜索后恢复当前状态过滤下的结果。
- 搜索激活时，命中分组临时展开，无命中分组隐藏。

无障碍：

- 输入框需要可读 label：`搜索标签页`。
- 清空按钮需要可读 label：`清空搜索`。

## StatusFilter

职责：

- 提供单一状态过滤。
- 搜索词可以叠加当前过滤。

选项：

- `全部`
- `打开`
- `归档`
- `重复`

样式：

- 使用 segmented control。
- 每个选项展示当前搜索条件下的计数。

示例：

```text
[全部 42] [打开 36] [归档 4] [重复 2]
```

行为：

- 同一时间只能选中一个状态。
- 状态过滤最多保留在当前侧边栏会话内，不跨浏览器重启。

## FeedbackRegion

职责：

- 展示非阻塞反馈。
- 不覆盖列表，不阻止用户继续操作。

类型：

- 操作失败
- API 读取失败后的重试提示
- 特殊 URL 不可归档提示

文案规则：

- 说明发生了什么。
- 说明用户能做什么。
- 不使用泛化文案，如 `出错了`。

示例：

```text
无法归档 chrome://extensions。特殊 URL 只能跳转或关闭。
```

```text
无法关闭这个标签页。它可能已经被关闭，请刷新列表。
```

## GroupList

职责：

- 展示按完整 host 分组后的标签清单。
- 承载空状态。

滚动：

- 只有 GroupList 滚动，顶部搜索和过滤保持可见。
- 分组标题可以普通滚动，不要求 sticky。MVP 先避免 sticky 复杂性。

排序：

- 分组排序第一版按组内总项数倒序；数量相同按 host 字母序。
- 组内打开项排在归档项前。
- 打开项按稳定窗口 id 和 tab index 排序，不按最近激活时间排序，避免点击后列表跳变。
- 归档项按最近归档时间倒序。

说明：

- MVP 采用“负载优先”的分组排序。标签页最多的域名最需要用户处理。

## GroupHeader

职责：

- 展示分组名称、数量摘要、折叠状态。
- 展示组级 favicon；同 host 的行共享分组 icon。
- 当分组内包含当前 active tab 时，展示轻量当前提示。
- 支持展开/折叠。

内容：

```text
▾ docs.google.com
8 项 · 打开 6 · 归档 1 · 重复 2
```

行为：

- 点击分组标题展开/折叠。
- 用户折叠状态持久化到 `chrome.storage.local`。
- 搜索或过滤激活时，命中分组临时展开。
- 退出搜索或过滤后恢复用户折叠状态。

无障碍：

- 分组标题是 button。
- 需要表达 expanded/collapsed 状态。

## InventoryRow

职责：

- 展示一个打开中的标签实例，或一个归档标签记录。
- 提供该行可用操作。

### 打开项

结构：

```text
│标题                              │
│docs.google.com/document/...       │
│当前 W2 重复 x2           归档 关闭│
```

字段：

- 标题
- 紧凑 URL：完整 host + path 截断
- 当前 active tab 标识
- 窗口标识
- 重复徽标
- 操作按钮

可用操作：

- 点击行跳转
- hover 或 focus 到行左缘选择命中区时显示从左侧滑入的悬浮选择控件；选中后控件停在同一终点位置常显，正文缩进进入批量选择态。
- 归档，特殊 URL 不显示或禁用
- 关闭

### 归档项

结构：

```text
│API notes                         │
│docs.google.com/document/...       │
│已归档 · 来自 W2             删除 │
```

字段：

- 归档时保存的标题
- 紧凑 URL
- 最后来源窗口
- 归档状态

可用操作：

- 点击行打开并恢复归档项
- hover 或 focus 到行左缘选择命中区时显示从左侧滑入的悬浮选择控件；选中后控件停在同一终点位置常显，正文缩进进入批量选择态。
- 删除归档记录

## 悬浮选择槽

职责：

- 在不牺牲列表横向空间的前提下提供批量选择入口。
- 让行和分组标题使用一致的选择语义。

规格：

- 默认不占据布局列，正文保持正常左边距。
- hover 或 focus 到行/分组标题左缘选择命中区时，显示小尺寸圆形选择控件和整行高度的矩形渐变遮罩；选择控件从左侧滑入，离开后滑出。
- 选中后选择控件在相同位置常显，不显示遮罩，正文左边距过渡到批量选择宽度；选择控件在缩进空间内保持左右间距一致。
- 分组部分选中时使用半选图标。

状态表达：

- 当前、打开、归档、重复和特殊 URL 通过 badge 与行底色表达。
- 选择控件只表达批量选择状态，不承担业务状态表达。

无障碍：

- 选择控件是可聚焦 button，并提供可读 label。
- 选中、半选和未选状态通过 `aria-pressed` 表达。
- 重复、归档、特殊 URL 必须有文字徽标或 label；颜色只是辅助。

## 当前页高亮

职责：

- 让用户知道当前 Chrome active tab 在清单中的位置。
- 切换 active tab 后自动把当前行滚动到列表可视区域。

行为：

- 当前行设置 `aria-current="page"`。
- 当前行展示 `当前` 徽标。
- 当前行使用轻量背景和边框突出。
- 当前行所属分组标题展示 `当前` 提示。
- 自动滚动只发生在当前项变化时，不因 hover 或 URL inspector 变化触发。

## URL Inspector

职责：

- 展示完整 URL。
- 替代 tooltip，避免长 URL 在窄侧边栏里被遮挡或截断。

行为：

- 默认固定在底部，展示最近预览 URL 或提示文案。
- hover 或 focus 行时，可以更新预览 URL。
- 点击行内 URL 后锁定 inspector 并展开。
- 锁定后 hover 其他行不覆盖内容。
- 展开状态下 URL 支持自动换行。
- 再次点击 inspector 可收起或展开。

## 操作按钮

MVP 操作按钮默认常显，不依赖 hover 才出现。按钮使用图标按钮，必须有可读 `aria-label`。tooltip 不作为按钮说明的主要机制。

按钮：

- 归档：`归档标签页`
- 关闭：`关闭标签页`
- 删除归档记录：`删除归档记录`

点击范围：

- 最小 28px × 28px。
- 行内视觉可以紧凑，但实际点击区域不能低于该尺寸。

按钮排列：

```text
打开项：归档 / 关闭
归档项：删除
特殊 URL：关闭
```

## EmptyState

### 没有普通窗口标签页

标题：

```text
没有可管理的普通窗口标签页
```

说明：

```text
打开一个普通 Chrome 窗口后，标签页会显示在这里。
```

### 搜索或过滤无结果

标题：

```text
没有匹配的标签页
```

说明：

```text
试试搜索域名、标题里的关键词，或清空当前过滤。
```

### 没有归档项

标题：

```text
还没有归档项
```

说明：

```text
归档一个普通网页后，它会出现在这里。
```

## LoadingState

首次加载文案：

```text
正在读取当前标签页...
```

展示方式：

- 顶部控制区可以先显示骨架。
- 列表区显示 4 到 6 条行骨架。
- 不使用大面积 spinner。

## ErrorState

主错误：

```text
无法读取标签页
Chrome 没有返回当前标签页列表。
[重试]
```

非阻塞错误：

- 出现在 FeedbackRegion 或 ToastRegion。
- 5 秒后自动消失。
- 用户也可以手动关闭。

## 文案表

| 场景 | 文案 |
| --- | --- |
| 搜索占位 | 搜索标题、域名或 URL |
| 全部过滤 | 全部 |
| 打开过滤 | 打开 |
| 归档过滤 | 归档 |
| 重复过滤 | 重复 |
| 重复徽标 | 重复 x{count} |
| 归档状态 | 已归档 |
| 来源窗口 | 来自 W{number} |
| 特殊 URL | 特殊 URL |
| 归档按钮 | 归档标签页 |
| 关闭按钮 | 关闭标签页 |
| 删除归档记录 | 删除归档记录 |
| 当前页 | 当前 |
| URL inspector 空状态 | 点击行内 URL 查看完整地址 |

说明：

- 状态徽标使用中文短标签，保持界面语言一致。
- `特殊 URL` 保留 URL 缩写。
- 操作按钮使用中文 `aria-label`，降低误操作。
