# 隐私与商店说明草案

本文档记录未来发布 Chrome 扩展时可使用的隐私说明和 Chrome Web Store 文案草案。当前只记录文档，不进入发布流程。

## 一句话描述

在 Chrome 侧边栏中查看、整理、归档和恢复当前打开的标签页。

## 简短描述

Chrome Tab Manager 帮助你在侧边栏中管理过多标签页：按完整域名查看当前普通窗口中的标签页，识别重复页面，跳转、关闭或归档暂时不用的页面，并在需要时恢复归档项。

## 详细描述草案

Chrome Tab Manager 是一个本地运行的标签页整理工具，面向经常同时打开多个窗口和大量标签页的用户。

MVP 功能包括：

- 在 Chrome 侧边栏查看所有普通窗口中的标签页。
- 按完整域名分组。
- 识别规范化 URL 相同的重复标签页。
- 跳转到指定标签页并聚焦所在窗口。
- 关闭单个标签页。
- 归档普通网页标签页，并在稍后恢复。
- 搜索标题、域名或 URL。
- 按全部、打开、归档、重复进行状态过滤。

扩展不管理隐身窗口，不记录完整浏览历史，不上传标签页数据。默认侧边栏模式不向网页注入内容；用户开启页面浮层后，扩展只在授权网页中注入重复提示 UI，不读取网页正文或表单内容。

## 权限说明

### `tabs`

用途：

用于读取当前打开标签页的标题、URL 和 favicon，识别重复标签页，并执行跳转、关闭、归档等本地标签页操作。

用户可见解释：

扩展需要知道当前打开了哪些标签页，才能在侧边栏中展示列表并执行管理操作。扩展不会保存完整浏览历史，也不会上传标签页数据。

### `storage`

用途：

用于在本机保存归档记录、分组折叠状态，以及当前浏览器会话内的待处理重复提示状态。

用户可见解释：

归档项和界面折叠状态保存在当前 Chrome profile 的本地存储中，不会通过扩展同步到其他设备。待处理重复提示只保存在当前浏览器会话内，浏览器重启后可以清理。

### `sidePanel`

用途：

用于在 Chrome 侧边栏显示标签页管理界面。

用户可见解释：

扩展的主要界面运行在 Chrome 侧边栏中。

### `scripting`（仅页面浮层开启后）

用途：

在用户主动开启并授权页面浮层后，用于在授权普通网页中展示重复页面提示浮层。页面浮层请求 `<all_urls>` 可选站点访问，以便重复提示能覆盖用户可能打开的普通网页；这不是默认安装权限。

用户可见解释：

页面浮层让用户在刚打开重复页面时直接选择跳转、保留或查看重复项。扩展不会读取网页正文、表单内容或登录态，也不会上传页面数据。

## 隐私承诺

MVP 隐私承诺：

- 不上传任何标签页数据。
- 不记录完整浏览历史。
- 默认不读取网页正文、DOM、表单内容、滚动位置或登录态。
- 默认不注入 content script。
- 默认不申请 host permissions。
- 用户开启页面浮层后，只在授权网页注入重复提示 UI，不读取网页正文或表单内容。
- 不管理隐身窗口。
- 不使用 `chrome.storage.sync`。
- 归档记录只保存在当前 Chrome profile 的本地存储中。

## 数据保存范围

扩展会在本机保存：

- 用户主动归档的 URL 记录。
- 归档记录的标题、favicon URL、原始 URL、规范化 URL。
- 最近归档时间和归档次数。
- 归档项的最后来源窗口标识。
- 普通分组折叠状态。
- 重复提示展示方式设置。
- 当前浏览器会话内最近一条待处理重复提示和已处理 tabId 集合。

扩展不会保存：

- 完整浏览历史。
- 网页正文。
- 表单内容。
- Cookie。
- 登录态。
- 页面截图。
- 页面滚动位置。
- 搜索词。
- 当前过滤状态的长期记录。
- 待处理重复提示的长期记录。

## 数据删除

用户可以在扩展中删除归档记录。删除归档记录只删除扩展本地保存的归档状态，不删除 Chrome 浏览器历史、书签或网页数据。

卸载扩展时，Chrome 会移除扩展本地存储中的数据。

## 发布前检查

发布前需要确认：

1. 默认安装权限仍然只包含侧边栏模式需要的权限。
2. 页面浮层相关 `scripting` 和 host access 使用用户主动开启后的可选授权。
3. content script 只用于页面浮层提示 UI，不读取页面内容。
4. 没有远程数据上传。
5. README、商店说明和隐私说明一致。
6. 测试计划覆盖权限和隐私边界。
