# UI 原型

本文档记录 MVP 阶段的 UI 原型方向。当前只产出设计文档和线框，不编写前端代码。

## 设计对象

设计对象是 Chrome 侧边栏里的标签页整理工作台。

目标用户是长期打开大量标签页、经常跨多个窗口工作的浏览器重度用户。这个界面的单一任务是：让用户快速判断当前打开了什么，并对单个标签页执行跳转、关闭、归档或恢复。

## 设计方向

关键词：密集、清晰、可扫描、低干扰。

这个 UI 不做营销式大标题，不做卡片堆叠，不做装饰性渐变。它更像一个窄幅调度台：左侧通过状态轨道提示每行的状态，右侧用克制的图标按钮提供动作。

主要视觉风险放在“状态轨道”上：每个列表行左侧有一条 3px 竖向状态轨道，用颜色和纹理区分打开、归档、重复、特殊 URL。它不是装饰，而是把用户最关心的状态编码进列表结构，让密集列表仍然可扫。

## 颜色

- `Panel` `#F7F8F3`：侧边栏背景，接近纸面但不偏暖黄。
- `Ink` `#202124`：主文本。
- `Muted` `#6B7280`：辅助文本、计数、路径。
- `Line` `#D8DDE4`：分隔线和边界。
- `Action Blue` `#2563EB`：跳转、当前焦点和可执行主动作。
- `Archive Green` `#2F7D57`：归档状态和恢复动作。
- `Duplicate Amber` `#B7791F`：重复标识。
- `Danger Red` `#B42318`：关闭、删除和错误。

调色避免单一蓝灰或深色控制台风格。背景保持安静，状态颜色只在轨道、徽标和关键图标上出现。

## 字体

- 界面字体：`Inter` / `PingFang SC` / system sans。用于标题、行文本、按钮。
- 数据字体：`JetBrains Mono` / `SF Mono`。用于 URL 片段、窗口编号、计数。
- 显示处理：不用大字号 display face。侧边栏是工具界面，个性来自结构和密度，不来自夸张标题。

建议字号：

- 顶部标题：15px / 20px，600
- 搜索输入：13px / 18px
- 行标题：13px / 18px，500
- 辅助 URL：11px / 15px，monospace
- 徽标和计数：10px / 12px，monospace

## 布局原则

侧边栏按 360px 宽度设计，最低兼容 320px。所有控件必须在窄宽度下保持可读，不依赖 hover 才能完成核心操作。

## MVP 主界面

```text
┌────────────────────────────────────┐
│ Tabs                         42    │
│ 当前普通窗口 · 归档已包含          │
├────────────────────────────────────┤
│ 🔎  搜索标题、域名或 URL            │
├────────────────────────────────────┤
│ [全部 42] [打开 36] [归档 4] [重复 2]│
├────────────────────────────────────┤
│ ▾ docs.google.com                  │
│   8 项 · 打开 6 · 归档 1 · 重复 2  │
├────────────────────────────────────┤
│┃  ▣ Quarterly roadmap              │
│┃  docs.google.com/document/...  W2 │
│┃  重复 x2          ↗   ◴   ×       │
├────────────────────────────────────┤
│┃  ▣ Quarterly roadmap              │
│┃  docs.google.com/document/...  W1 │
│┃  重复 x2          ↗   ◴   ×       │
├────────────────────────────────────┤
│╎  ▣ API notes                      │
│╎  docs.google.com/document/...     │
│╎  已归档 · 来自 W2    ↗   🗑        │
├────────────────────────────────────┤
│ ▾ github.com                       │
│   5 项 · 打开 5 · 归档 0 · 重复 0  │
├────────────────────────────────────┤
│┃  ▣ Pull request #48               │
│┃  github.com/org/repo/pull/48  W1  │
│┃                      ↗   ◴   ×    │
└────────────────────────────────────┘
```

说明：

- `┃` 表示打开项状态轨道。
- `╎` 表示归档项状态轨道。
- 重复项使用 `重复 x2` 徽标，不合并行。
- `W1`、`W2` 是最小窗口标识。
- `↗` 是跳转或恢复。
- `◴` 是归档。
- `×` 是关闭。
- `🗑` 是删除归档记录。

实际实现时优先使用图标库中的一致图标，不依赖这些字符本身。

## 行结构

```text
┌────────────────────────────────────┐
│轨 favicon 标题                 Wn  │
│轨 host/path compact                │
│轨 status badge        actions      │
└────────────────────────────────────┘
```

行高建议：

- 普通行：56px
- 有重复徽标或归档来源时：64px
- 分组标题：44px

动作按钮：

- 操作按钮默认常显，不依赖 hover 才出现。
- 跳转：始终可见。
- 归档：只对普通网页打开项可见。
- 关闭：只对打开项可见。
- 删除归档记录：只对归档项可见。

特殊 URL 行：

```text
│▥  Chrome Extensions             W1 │
│▥  chrome://extensions              │
│▥  特殊 URL           ↗       ×     │
```

特殊 URL 可跳转、可关闭、不可归档。

## 过滤状态

```text
┌────────────────────────────────────┐
│ Tabs                         42    │
├────────────────────────────────────┤
│ 🔎  roadmap                         │
├────────────────────────────────────┤
│ [全部 3] [打开 2] [归档 1] [重复 2] │
├────────────────────────────────────┤
│ ▾ docs.google.com                  │
│   3 项 · 打开 2 · 归档 1 · 重复 2  │
│ ...命中项...                       │
└────────────────────────────────────┘
```

搜索或过滤激活时：

- 只展示命中分组。
- 命中分组临时展开。
- 退出搜索或过滤后恢复用户折叠状态。

## 空状态

### 搜索无结果

```text
┌────────────────────────────────────┐
│ Tabs                         42    │
├────────────────────────────────────┤
│ 🔎  billing webhook                 │
├────────────────────────────────────┤
│ [全部 0] [打开 0] [归档 0] [重复 0] │
├────────────────────────────────────┤
│ 没有匹配的标签页                   │
│ 试试搜索域名、标题里的关键词，      │
│ 或清空当前过滤。                   │
└────────────────────────────────────┘
```

### 没有归档项

```text
┌────────────────────────────────────┐
│ Tabs                         36    │
├────────────────────────────────────┤
│ [全部 36] [打开 36] [归档 0] [重复 2]│
├────────────────────────────────────┤
│ 还没有归档项                       │
│ 归档一个普通网页后，它会出现在这里。│
└────────────────────────────────────┘
```

### API 读取失败

```text
┌────────────────────────────────────┐
│ Tabs                               │
├────────────────────────────────────┤
│ 无法读取标签页                     │
│ Chrome 没有返回当前标签页列表。     │
│ [重试]                             │
└────────────────────────────────────┘
```

错误文本要说明发生了什么和用户能做什么，不使用含糊的“出错了”。

## 交互规则

### 跳转

点击打开项的跳转按钮后，Chrome 激活该标签实例并聚焦所在窗口。

点击归档项的跳转按钮后，在当前普通窗口打开保存的原始 URL，并移除归档状态。

### 关闭

关闭只移除打开中的标签实例，不创建归档记录。MVP 不做二次确认，不提供撤销。

### 归档

归档只对普通网页打开项可用。归档会关闭对应标签实例；如果同一规范化 URL 没有其他打开实例，则创建归档状态。MVP 不做二次确认，不提供撤销。

### 删除归档记录

删除归档记录只移除扩展内归档状态，不影响浏览器历史或书签。

## 自检

这套方向刻意避开了常见模板：

- 没有使用大面积深色控制台风格。
- 没有使用暖米色杂志风格。
- 没有用营销页式 hero、统计卡片或装饰渐变。
- 视觉记忆点来自状态轨道，而不是无意义装饰。

风险点：

- 状态轨道如果颜色过重，会让窄侧边栏显得吵。实现时轨道宽度应控制在 3px，颜色只在关键状态出现。
- 图标按钮密度高，必须保证可点击面积和可读标签。
- URL 辅助文本要截断，但完整 URL 必须能通过详情或悬浮提示查看。

## MVP 之后暂不进入原型的能力

以下能力不进入 MVP 原型主流程：

- 自定义分组配置
- 侧边栏置顶和置顶区
- 批量操作
- 新页面重复提示
- 休眠自动归档
- 内存指标
- 独立设置页
