# 实施计划

状态：MVP 已完成，当前计划从 MVP 后置功能阶段继续推进。

本文档记录从当前代码基线出发的阶段计划。已完成的 MVP 细节见 [MVP 完成检查](./mvp-completion.md)，不要在这里重复维护旧的阶段 0 启动清单。

## 已完成：MVP

MVP 已覆盖：

- MV3 + Vite + React + TypeScript + shadcn/ui 扩展骨架。
- side panel 标签清单。
- 普通窗口 snapshot 读取，排除隐身窗口。
- 完整 host 分组和分组折叠持久化。
- 规范化 URL 和重复识别。
- 搜索与全部、打开、归档、重复过滤。
- 跳转、关闭、归档、恢复归档、删除归档记录。
- loading、empty、error 和非阻塞操作反馈。
- 当前 active tab 高亮和可视锚定。
- 底部固定 URL inspector。

## 下一阶段原则

- 每个阶段完成后更新对应产品、技术、测试文档。
- 新增 Chrome 权限前先更新 [Chrome 权限最小集](./permissions-plan.md) 和 [风险清单](./risk-register.md)。
- 新增持久化字段前先更新 [Storage Schema](./storage-schema.md)。
- 新增运行时状态前先更新 [Runtime State Model](./runtime-state-model.md)。

## 阶段 1：自定义分组

目标：

在默认完整 host 分组之上，支持用户配置自定义分组。

范围：

- 自定义分组 storage schema。
- 完整 host 匹配规则。
- 规范化 URL 正则匹配规则。
- 分组优先级：有序列表，首个命中胜出。
- 分组和规则启用/禁用。
- 无效正则处理，不破坏标签清单。
- 基础配置 UI。

验收标准：

- 命中自定义分组的标签进入自定义分组。
- 未命中自定义分组时回落完整 host 分组。
- 多条规则任意命中即可进入该分组。
- 禁用分组或规则后不参与匹配，但配置保留。
- 无效正则有明确错误，不影响其他规则和列表渲染。
- 单元测试覆盖 host 匹配、URL 正则、优先级和 fallback。

## 阶段 2：侧边栏置顶

目标：

实现扩展自己的 URL 级置顶能力，不改变 Chrome 原生 pinned tab。

范围：

- URL 级置顶 storage schema。
- 置顶区。
- 置顶项手动排序。
- 新置顶项默认放在最前。
- 置顶项不在普通分组列表中重复展示。

验收标准：

- 有打开实例时，置顶区展示对应标签实例。
- 没有打开实例但存在归档状态时，置顶区展示归档记录。
- 置顶状态跨 active/archived 状态保留。
- 普通列表中不重复展示置顶项。

## 阶段 3：重复新开页处理

目标：

当用户打开已存在的规范化 URL 时，根据配置提示或自动处理。

范围：

- 重复处理配置：提示模式和自动模式。
- 默认提示模式。
- 待处理重复提示的运行时状态。
- 提示中支持跳转到已有实例、保留当前实例、查看重复实例。
- 自动模式跳转到最近激活的已有实例，并移除新重复标签页。

验收标准：

- 新打开重复 URL 时能产生提示。
- 侧边栏未打开时，提示模式不主动打开侧边栏。
- 自动模式能完成跳转和移除新重复 tab。
- 保留当前重复标签页只对当次生效，不产生长期豁免。

## 阶段 4：休眠与自动归档

目标：

识别长时间未激活标签页，并在用户开启后自动归档。

范围：

- 标签实例最后激活时间。
- 全局休眠阈值，默认七天。
- 实例级保活。
- 自动归档开关，默认关闭。
- 自动归档跳过保活、音频播放、Chrome 原生置顶实例。
- 最近自动归档记录。

验收标准：

- 扩展首次观察到已有标签页时，不立即判定为休眠。
- 用户未开启自动归档时不自动关闭任何标签页。
- 保活只作用于具体 tab instance，实例关闭后结束。
- 自动归档结果在侧边栏中可见。

## 阶段 5：批量操作

目标：

在已有单项操作稳定后，加入选择模型和批量操作。

范围：

- 多选状态。
- 分组级选择当前可见项。
- 批量关闭。
- 批量归档。
- 批量删除归档记录。
- 高风险摘要和确认。

验收标准：

- 分组级选择尊重当前搜索和过滤范围。
- 批量删除归档记录只作用于归档项，并跳过打开项。
- 批量归档跳过特殊 URL。
- 高风险项在执行前有明确摘要。

## 阶段 6：按需诊断指标

目标：

按需展示标签页相关内存指标，不默认持续采样。

范围：

- 评估并申请 `chrome.processes` 所需权限。
- 标签实例到浏览器进程的按需查询。
- 进程级内存指标展示。
- 指标说明文案。

验收标准：

- 用户主动刷新或打开详情时才采集内存指标。
- 多个标签页共享进程时，UI 不把指标误导为单标签独占值。
- 关闭内存指标展示后，不继续采样。

## 阶段 7：发布前打磨

目标：

准备更完整的真实 Chrome 回归和商店发布材料。

范围：

- 真实 Chrome 手动回归记录。
- 大量标签页性能检查。
- service worker 休眠和恢复后的状态一致性检查。
- 权限说明和隐私文案核对。
- Chrome Web Store 素材准备。

验收标准：

- 测试计划覆盖 MVP 和已完成后置功能。
- 权限申请与产品能力一致。
- 隐私说明和实际行为一致。
