# 实施计划

当前计划只拆分工作，不编写代码。

## 阶段 0：项目初始化

- 创建 Chrome Extension MV3 项目骨架。
- 配置 TypeScript、React、Vite。
- 初始化 shadcn/ui。
- 按 MVP 需要添加 shadcn/ui 组件。
- 配置基础 lint、format、类型检查。
- 创建 side panel、service worker、共享类型目录。

验收标准：

- 扩展可以在 Chrome 开发者模式加载。
- 侧边栏可以打开并显示静态页面。
- service worker 可以启动并输出最小诊断信息。
- shadcn/ui 已按项目配置初始化，MVP 所需组件可正常导入。

## 阶段 1：实时标签清单

- 读取所有普通窗口的标签实例。
- 排除隐身窗口。
- 读取窗口信息和 Chrome 原生 Tab Group 信息。
- 计算规范化 URL、完整 host、重复组。
- 在侧边栏展示标签清单。
- 展示分组数量摘要。
- 记住普通分组折叠状态。
- 实现标签实例行的标题优先展示。
- 实现最小 loading、error 和操作失败提示。
- 实现基础无障碍要求。

验收标准：

- 侧边栏能展示所有普通窗口中的标签实例。
- 同一规范化 URL 的重复标签实例显示重复标识和数量。
- 窗口和浏览器标签组可作为元信息展示。
- 分组标题显示总项数、打开项数、归档项数和重复项数。
- 普通分组默认展开，新分组默认展开，用户折叠状态可保留。
- 搜索或过滤激活时，命中分组临时展开；退出后恢复原折叠状态。
- 标签实例行主文本为标题，辅助文本为完整域名和路径紧凑形式，完整 URL 可查看但不直接占据主列表。
- 首次加载、API 读取失败和操作失败都有明确状态反馈。
- 操作按钮有可读标签，主要交互可键盘聚焦，状态不只依赖颜色表达。

## 阶段 2：基础操作

- 实现跳转到打开中标签实例。
- 实现关闭单个标签实例。
- 实现批量关闭。
- 实现批量操作选择模型。
- 实现分组级选择当前可见项。
- 实现搜索和基础过滤。

验收标准：

- 点击标签实例能聚焦对应窗口并激活标签页。
- 关闭动作不创建归档记录。
- 批量关闭只作用于用户选中的项。
- 分组级选择尊重当前搜索和过滤范围，只选择可见项。

## 阶段 3：归档

- 实现单个标签实例归档。
- 实现批量归档。
- 实现归档标签记录展示。
- 实现从归档记录跳转并移除归档状态。
- 实现删除归档记录。
- 实现批量删除归档记录。
- 实现同一规范化 URL active 与 archived 互斥。
- 实现批量归档安全规则。
- 实现批量关闭安全规则。
- 实现单个关闭和归档确认规则。

验收标准：

- 归档会关闭对应标签实例。
- 同一 URL 仍有其他打开实例时，不创建归档状态。
- 没有剩余打开实例时，创建归档状态。
- 打开归档项后，归档状态消失。
- 删除归档记录只删除扩展内归档状态，不影响浏览器历史或书签。
- 批量删除归档记录只作用于归档项，并跳过打开项。
- 批量归档跳过特殊 URL；音频播放和 Chrome 原生置顶实例需要二次确认；保活不阻止手动归档。
- 批量关闭允许关闭特殊 URL 但计入确认摘要；音频播放、Chrome 原生置顶、保活、最近五分钟激活过的实例需要二次确认。
- 单个关闭或归档默认不确认；音频播放和 Chrome 原生置顶实例需要确认；特殊 URL 不允许归档但允许关闭。

## 阶段 4：分组系统

- 实现完整 host 默认分组。
- 实现自定义分组配置。
- 支持完整 host 匹配。
- 支持对规范化 URL 的正则匹配。
- 实现自定义分组优先级和首个命中胜出。
- 实现每个自定义分组包含多条规则，组内任意规则命中。
- 实现自定义分组和规则的启用/禁用。

验收标准：

- 标签清单优先使用自定义分组。
- 未命中自定义分组时回落完整 host。
- 无效正则不会破坏整体列表。
- 禁用的自定义分组或规则不参与匹配，但配置保留。

## 阶段 5：侧边栏置顶

- 实现 URL 级侧边栏置顶。
- 实现置顶区。
- 实现置顶区手动排序。
- 实现新置顶项默认放在最前。
- 确保置顶项不在普通分组列表中重复展示。

验收标准：

- 有打开实例时，置顶区展示对应实例。
- 没有打开实例但存在归档状态时，置顶区展示归档记录。
- 置顶状态跨 active/archived 状态保留。

## 阶段 6：重复处理

- 实现提示模式。
- 实现自动模式。
- 默认使用提示模式。
- 多个既有重复实例时，默认选择最近激活的实例。
- 支持当次保留当前重复标签页。

验收标准：

- 新打开重复 URL 时能触发提示。
- 自动模式能跳转到最近激活的既有实例，并移除新重复实例。
- 当次保留不产生长期豁免。

## 阶段 7：休眠与自动归档

- 记录标签实例最后激活时间。
- 标记超过阈值的休眠标签页。
- 实现实例级保活。
- 实现休眠自动归档开关，默认关闭。
- 自动归档跳过保活、音频播放、Chrome 原生置顶实例。
- 实现最近自动归档记录展示。

验收标准：

- 默认阈值为七天。
- 扩展首次观察到已有标签页时，不会立即将其判定为休眠。
- 保活只作用于具体标签实例，实例关闭后结束。
- 自动归档只在用户开启后执行。
- 自动归档发生后，侧边栏能展示近期自动归档结果。

## 阶段 8：打磨与验证

- 补充关键单元测试。
- 补充 Chrome 扩展手动验证清单。
- 验证大量标签页下侧边栏性能。
- 验证 service worker 休眠和恢复后的状态一致性。
- 检查权限最小化和隐私说明。

验收标准：

- 核心状态规则有测试覆盖。
- 手动验证清单覆盖打开、关闭、归档、重复、置顶、分组、休眠自动归档。
- 权限申请与产品能力一致。

## 阶段 9：按需诊断指标

- 评估并申请 `chrome.processes` 所需权限。
- 实现标签实例到浏览器进程的按需查询。
- 展示进程级内存指标。
- 避免默认持续轮询内存。
- 在 UI 文案中说明该指标不是单个标签页的精确独占内存。

验收标准：

- 用户主动刷新或打开详情时才采集内存指标。
- 多个标签页共享进程时，UI 不把指标误导为单标签独占值。
- 关闭内存指标展示后，不继续采样。
