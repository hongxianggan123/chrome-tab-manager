# 测试计划

本文档记录 MVP 阶段的测试计划。Chrome 扩展行为依赖真实浏览器事件，MVP 需要同时覆盖手动验证和关键纯逻辑单元测试。

## MVP 手动验证

### 加载扩展

步骤：

1. 在 Chrome 开发者模式加载扩展。
2. 打开扩展侧边栏。

预期：

- 侧边栏可以打开。
- 首次加载时有 loading 状态。
- 加载完成后显示当前普通窗口中的标签页。

### 普通窗口与隐身窗口

步骤：

1. 打开两个普通窗口，并分别打开多个标签页。
2. 打开一个隐身窗口，并打开多个标签页。
3. 打开侧边栏。

预期：

- 侧边栏显示所有普通窗口中的标签实例。
- 侧边栏不显示隐身窗口中的标签页。
- 标签实例显示最小窗口标识。

### 默认完整 host 分组

步骤：

1. 打开 `docs.google.com` 和 `mail.google.com`。
2. 打开侧边栏。

预期：

- 两个标签页分属不同完整 host 分组。
- 分组标题显示总项数、打开项数、归档项数、重复项数。

### 重复识别

步骤：

1. 打开 `https://example.com/a?x=1#one`。
2. 打开 `https://example.com/a?x=1#two`。
3. 打开 `https://example.com/a?x=2`。

预期：

- 前两个标签页被识别为重复。
- 第三个标签页不属于该重复组。
- 重复标签实例仍分行展示，并显示共享重复标识和数量。

### 重复提示触发范围

步骤：

1. 已有一个普通网页标签页打开 `https://example.com/a?x=1`。
2. 新建一个普通网页标签页并导航到 `https://example.com/a?x=1#two`。
3. 对其中一个重复标签页执行 reload 或 hash-only 变化。

预期：

- 新建标签页导航稳定后产生重复提示或按重复处理配置执行。
- reload 和 hash-only 变化只刷新重复识别状态，不产生新的重复提示。
- 侧边栏未打开时，连续产生多个重复提示只保留最近一条待处理提示。
- 用户在提示中选择跳转到已有标签页时，目标既有标签页被激活，触发提示的新重复标签页被关闭。
- 触发提示的新重复标签页即使正在播放音频或 Chrome 原生 pinned，也与普通标签页保持相同行为。
- 用户在提示中选择保留当前标签页时，提示被清理，该标签实例不再反复提示，但仍作为重复标签页留在标签清单中。
- 重复提示展示在顶部控制区下方、标签清单上方。
- 提示中的跳转、保留、查看重复三个动作执行后都关闭提示。
- 重复提示展示 30 秒后自动关闭，最后 5 秒在保留按钮上展示倒计时；自动关闭不关闭新重复标签页，并且同一个标签实例不再反复提示。
- 默认展示方式为侧边栏提示，不要求页面注入授权。
- 用户开启页面浮层并授予 `<all_urls>` 可选站点访问后，普通网页中的重复提示以页面浮层展示。
- 页面浮层只在产生重复提示的新重复标签实例中按需动态注入，不在所有页面静态常驻。
- 页面浮层固定在页面右上角，不可拖动；窄屏下不超出视口。
- 页面浮层使用 Shadow DOM 隔离样式，按钮可键盘聚焦，动效尊重 `prefers-reduced-motion`。
- 页面浮层中的跳转、保留、查看重复三个动作与侧边栏提示语义一致。
- 点击 `查看重复` 后打开或切换到侧边栏，清空搜索，切到重复过滤，并优先定位触发提示的新重复标签页；如果它已不存在，则定位默认目标。
- 用户拒绝页面浮层授权、授权被撤销、注入失败或页面不可注入时，降级为侧边栏待处理提示和扩展图标轻提醒。
- 页面浮层授权被用户关闭或撤销后，设置自动回到侧边栏，并显示非阻塞提示。
- service worker 挂起并恢复后，最近一条待处理重复提示仍可从 `chrome.storage.session` 恢复；已处理 tabId 不会再次提示。
- 侧边栏标题区的设置入口能进入设置视图，设置视图替换主列表区域并提供返回。
- 设置面板按分区展示当前重复提示配置，并能未来扩展其他配置项。
- 返回标签清单后，原有搜索、过滤和选择状态仍保留。
- 切换重复提示展示方式即时生效；页面浮层授权失败时保持或回退到侧边栏展示，并显示非阻塞错误。

### 跳转

步骤：

1. 在多个普通窗口中打开标签页。
2. 在侧边栏点击某个非当前窗口中的标签实例。

预期：

- Chrome 聚焦该标签页所在窗口。
- 目标标签页被激活。

### 单个关闭

步骤：

1. 在侧边栏点击某个标签实例的关闭操作。

预期：

- 对应 Chrome 标签页关闭。
- 不创建归档记录。
- MVP 不出现二次确认。
- MVP 不提供撤销。

### 单个归档

步骤：

1. 在侧边栏归档一个普通网页标签实例。

预期：

- 对应 Chrome 标签页关闭。
- 如果同一规范化 URL 没有其他打开实例，侧边栏出现归档项。
- 归档项保存标题、favicon、原始 URL、规范化 URL、最后来源窗口。
- MVP 不出现二次确认。
- MVP 不提供撤销。

### 同 URL 多实例归档

步骤：

1. 打开两个规范化 URL 相同的标签实例。
2. 归档其中一个实例。

预期：

- 被归档的实例关闭。
- 因为同一规范化 URL 仍有打开实例，不创建归档状态。

### 特殊 URL

步骤：

1. 打开 `chrome://extensions`。
2. 在侧边栏查看并尝试操作。

预期：

- 特殊 URL 可以展示和跳转。
- 特殊 URL 不允许归档。
- 特殊 URL 可以关闭。

### 恢复归档

步骤：

1. 归档一个普通网页。
2. 点击归档项。

预期：

- 在当前普通窗口打开归档项保存的原始 URL。
- 归档状态消失。
- 不强制回到最后来源窗口。

### 删除归档记录

步骤：

1. 创建一个归档项。
2. 删除该归档记录。

预期：

- 归档项从侧边栏消失。
- 浏览器历史和书签不受影响。

### 搜索

步骤：

1. 创建多个打开项和归档项。
2. 输入能命中标题、完整 host 或规范化 URL 的搜索词。

预期：

- 搜索同时覆盖打开项和归档项。
- 命中分组临时展开。
- 无命中分组隐藏。
- 清空搜索后恢复原分组折叠状态。

### 状态过滤

步骤：

1. 在侧边栏切换全部、打开、归档、重复过滤。
2. 输入搜索词。

预期：

- MVP 只支持单一状态过滤。
- 搜索可以叠加当前状态过滤。
- 状态过滤最多保留在当前侧边栏会话内，不跨重启。
- 搜索词不持久化。

### Side panel 推送刷新

步骤：

1. side panel 未连接时触发 Chrome tabs/windows 事件。
2. 之后打开 side panel 并建立 runtime port。
3. side panel 打开期间模拟 runtime port 断开并重连。

预期：

- service worker 保留 dirty 标记，port 连接后补发最新 `state:changed`。
- side panel port 断开后会自动重连。
- 重连后 side panel 重新请求 `state:get`，补齐断开期间可能漏掉的标签页变化。

### 空状态

步骤：

1. 触发没有普通窗口标签页、搜索无结果、归档过滤无归档项等状态。

预期：

- 侧边栏展示可理解的空状态，不显示空白界面。

### 错误状态

步骤：

1. 模拟或触发 Chrome API 读取失败、标签已被关闭后再操作等错误。

预期：

- 侧边栏显示非阻塞错误提示。
- 整个侧边栏不因单个失败操作不可用。

### 基础无障碍

步骤：

1. 使用键盘 Tab 在主要控件间移动。
2. 使用辅助技术或浏览器可访问性检查查看按钮标签。

预期：

- 主要操作可聚焦。
- 操作按钮有可读标签。
- 重复、归档、错误、空状态不只依赖颜色表达。

## 单元测试重点

### URL 规范化

- 保留 origin、path、query。
- 忽略 fragment。
- query 不同时不视为同一规范化 URL。
- hash 不同时视为同一规范化 URL。

### 重复组计算

- 同一规范化 URL 的打开实例形成重复组。
- 只有一个实例时不显示重复。
- 归档项不参与重复组。

### 归档状态

- 同一规范化 URL 有剩余打开实例时，不创建归档状态。
- 没有剩余打开实例时，创建归档状态。
- 打开归档项后移除归档状态。
- 删除归档记录只移除扩展内状态。

### 分组

- 默认按完整 hostname 分组。
- 不同子域名是不同分组。
- 特殊 URL 有稳定展示分组。

### 搜索与过滤

- 搜索匹配标题、完整 host、规范化 URL。
- 搜索覆盖打开项和归档项。
- 单一状态过滤正确筛选全部、打开、归档、重复。
- 搜索和当前状态过滤可叠加。

### 视图状态

- 分组折叠状态可持久化。
- 搜索词不持久化。
- 状态过滤不跨浏览器重启持久化。

## MVP 后置：批量操作验证

### 手动验证

步骤：

1. 使用标签行左侧选择槽选择多个打开项和归档项。
2. 使用分组标题左侧选择槽选择当前可见项。
3. 分别触发批量关闭、批量归档和批量删除归档记录。

预期：

- 行级选择槽不改变行点击跳转或恢复的主行为。
- 行级选择和分组级选择状态一致。
- 分组级选择只作用于当前搜索和过滤后的可见项。
- 批量关闭只关闭打开项，归档项跳过并在确认摘要中说明。
- 批量归档只处理普通网页打开项，特殊 URL 和归档项跳过并在确认摘要中说明。
- 批量删除归档记录只删除归档项，打开项跳过并在确认摘要中说明。
- 正在播放音频、Chrome 原生置顶、特殊 URL 和最近使用项会出现在高风险摘要中。

### 单元测试重点

- 批量归档计划正确跳过特殊 URL 和归档项。
- 批量删除归档记录计划正确跳过打开项。
- 批量关闭计划保留特殊 URL 目标，并在风险摘要中标识。
